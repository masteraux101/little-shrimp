# Test Session Prerequisites

## Required Environment Variables
These must be set before running any test scripts:

```bash
export GEMINI_KEY="<your-actual-gemini-api-key>"
export GITHUB_PAT="<your-github-pat>"
export GITHUB_OWNER="masteraux101"
export GITHUB_REPO="my-agent-session"
```

## Session Configuration Requirements

When creating a test session via Playwright, the following fields MUST be filled:

### Required Fields (for any session):
- **API Key** (`#set-api-key`): Use `process.env.GEMINI_KEY`
  - ❌ DON'T use placeholder values like 'test-key'
  - ✅ DO use actual API key from environment

- **Model Name** (`#set-model`): Must be set to a valid model
  - Example: `gemini-2.5-flash`
  - This field is REQUIRED or session will show toast error

- **Passphrase** (`#set-passphrase`): Required for new sessions
  - Example: `test-pass`
  - Used to encrypt session data

### Recommended DOM-based Fill Method
Instead of using `.fill()` or `.selectOption()`, use direct DOM manipulation:

```javascript
await page.evaluate(({key, model, pass}) => {
  document.querySelector('#set-api-key').value = key;
  document.querySelector('#set-model').value = model;
  document.querySelector('#set-passphrase').value = pass;
}, {key: GEMINI_KEY, model: 'gemini-2.5-flash', pass: 'test-pass'});
```

## Common Issues & Fixes

### Issue: "Please set a model in session settings"
**Cause**: Model field (`#set-model`) was not filled
**Fix**: Check that model value is correctly set before clicking apply

### Issue: Command doesn't execute
**Cause**: Session creation didn't complete properly
**Fix**: Increase `waitForTimeout()` after `#apply-settings` click (min 1500ms)

### Issue: CORS errors when loading from GitHub
**Expected**: This is normal, our code has automatic CORS proxy fallback
**Status**: Check browser console, should still load successfully

## Testing SOUL and SKILL Loading

### /soul command
```
/soul <built-in-name>     # e.g., /soul BrowserAgent Assistant
/soul <github-raw-url>    # e.g., /soul https://raw.githubusercontent.com/.../SOUL.md
```

### /skill command
```
/skill <built-in-name>    # e.g., /skill github-scheduler
/skill <github-raw-url>   # e.g., /skill https://raw.githubusercontent.com/.../skill.md
```

Both support external URLs and will automatically use CORS proxy if needed.
